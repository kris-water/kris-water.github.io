###### 知识扩展：为什么 JAVA 8 会很火，但是并不是 7 或者 9，JAVA8的后续版本有比较流行的么？

11 17 21 都是LTS版本，且支持周期均超过2030年

| Java 版本     | 特点描述                                                                                                                         |
| :---------- | :--------------------------------------------------------------------------------------------------------------------------- |
| **Java 7**  | 首先因为 7 和 9 并不是 LTS 版本，而且 7 的改动相比于 6 太少                                                                                       |
| **Java 8**  | 现在除了 JAVA 8 比较流行的版本，首先一定是 LTS 版本的                                                                                            |
| **Java 9**  | 9 是因为改动的过于大，风险过高，且不是 LTS，升级后没有保障                                                                                             |
| **Java 11** | JAVA 11（局部变量类型推断（var 定义，内部结构可用，左侧根据右侧类型退队），http client（官方支持 HTTP1.2,2.0 版本，原生支持 WS））                                         |
| **Java 17** | JAVA 17 密封类，新模式匹配（预览）                                                                                                        |
| **Java 21** | JAVA 21 虚拟线程（传统线程在遇到 IO 阻塞或 sleep 等，会阻塞。虚拟线程是用户态线程会在被阻塞时被内核态进程切换，执行其他任务）。现在比较流行的是 JAVA 21 ，因为虚拟线程这个特性提升很大，JAVA 8 的项目数量在逐年下降。 |
